## 📋 摘要

Q2 产品路线图规划会确定了两个核心方向：用户增长（优化 onboarding 流程，目标 7 日留存从 23% 提升至 35%）和技术债务清理（支付模块微服务化拆分）。数据平台建设做前期调研和架构设计，Q3 全面实施。

## 📝 议题

### 1. 用户增长策略
- 新用户 7 日留存仅 23%，主要流失在引导流程
- 方案：将 7 步引导压缩至 3 步，采用渐进式引导
- 预期效果：留存提升至 35%，DAU 增加约 8 万

### 2. 技术债务清理
- 支付模块 15000 行单体代码，Q3 需接入 3 个新渠道
- 方案：按渠道逐步拆分，先拆支付宝通道
- 前端并行升级 React 16→18

### 3. 数据平台建设
- 当前数据分散在 GA、自建 BI、运营后台三个系统
- 方案：统一数据仓库 + dbt + Metabase
- Q2 做调研和架构设计，Q3 全面实施

### 4. 优先级讨论
- 资源只够 Q2 做两个方向
- 结论：用户增长 + 技术债务并行，数据平台做前期准备

## ✅ 决策

- **Q2 核心方向**: 用户增长 + 技术债务并行
  - 理由：直接影响 OKR 和 Q3 业务计划
- **数据平台节奏**: Q2 调研设计，Q3 实施
  - 理由：资源有限，可接受延迟
